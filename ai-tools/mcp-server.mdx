## B2Bware MCP Server

The B2Bware MCP Server is a hosted remote service that connects AI assistants to your live DataHub using the [Model Context Protocol (MCP)](https://modelcontextprotocol.io). No local installation is required: configure your API key and interact with products, customers, orders, carts, checkout, and tax through natural language.

**Server URL:** `https://mcp.b2bware.com/sse`

<Callout kind="info" title="What is MCP?">
The Model Context Protocol (MCP) is an open standard that lets AI assistants call external tools and access data sources. With the B2Bware MCP Server connected, your assistant can browse the product catalog, search customers, manage carts, preview and complete checkout, calculate tax, and create or update orders — all in conversation.
</Callout>

## Available Tools

The MCP Server exposes **27 tools** for products, categories, orders, carts, checkout, tax, customers, and authentication. Order responses include a synthetic `order_number` (e.g. `ORD-2024-00001`); the status value `"processing"` is normalized from the API’s internal spelling.

<Tabs>
  <Tab title="Products" icon="box">

| Tool | Description |
|------|-------------|
| `list_products` | List products with pagination, sort, and filters (e.g. category, anchor_category_id, product_type, exclude_configurable, exclude_child_products) |
| `search_products` | Full-text search by name, SKU, description; use `exclude_configurable=true` to return only orderable products |
| `get_product` | Get a single product by ID with details (price, media, categories) |
| `list_categories` | List product categories (tree) with pagination |
| `get_category` | Get a single category by ID |

  </Tab>
  <Tab title="Customers" icon="users">

| Tool | Description |
|------|-------------|
| `list_customers` | List customers with filter options (email, customer_number, pagination) |
| `search_customers` | Search customers by name, email or company |
| `get_customer` | Retrieve customer profile with shipping and billing addresses by ID |
| `create_customer` | Create a new customer account |
| `update_customer` | Update an existing customer (name, phone, status, etc.) |
| `list_customer_addresses` | List all shipping and billing addresses for a customer |
| `create_shipping_address` | Create a shipping address for a customer |
| `create_billing_address` | Create a billing address for a customer |

  </Tab>
  <Tab title="Orders" icon="shopping-cart">

| Tool | Description |
|------|-------------|
| `list_orders` | List orders with filters (customer, status, date range, price) and sorting; use `sort=-id` for newest first |
| `get_order` | Retrieve order details with line items by ID |
| `update_order_status` | Update the status of an order (pending, processing, completed, cancelled) |

  </Tab>
  <Tab title="Cart & Checkout" icon="credit-card">

| Tool | Description |
|------|-------------|
| `upsert_cart` | Create or update a cart by `session_uuid`; pass `cart_items` (product_id, quantity, optional price) in one call |

| `list_carts` | List all carts, optionally filtered by customer ID |

## Getting Your API Key

<Steps>
  <Step title="Log in to B2Bware" icon="log-in">
    Go to your B2Bware admin panel.
  </Step>
  <Step title="Navigate to API settings" icon="settings">
    Go to **Settings → API → API Keys**.
  </Step>
  <Step title="Create or copy your API key" icon="key">
    Create a new API key or copy an existing one. This key works for both the REST API and the MCP Server.
  </Step>
</Steps>

## Authentication Methods

<Tabs>
  <Tab title="Header-Based (Recommended)" icon="shield">

Pass your API key via the `Authorization` header:

```
URL: https://mcp.b2bware.com/sse
Header: Authorization: Bearer YOUR_B2BWARE_API_KEY
```

<Callout kind="tip">
The header approach is more secure since the API key does not appear in server logs or browser history.
</Callout>

  </Tab>
  <Tab title="URL Parameter" icon="link">

For clients that do not support custom headers, pass the API key as a query parameter:

```
https://mcp.b2bware.com/sse?key=YOUR_B2BWARE_API_KEY
```

Use URL parameters only when custom headers are not supported by your client.

  </Tab>
</Tabs>

## Client Configuration

<Tabs>
  <Tab title="Cursor" icon="code">

**File:** `~/.cursor/mcp.json` (Mac) or `%USERPROFILE%\.cursor\mcp.json` (Windows)

```json
{
  "mcpServers": {
    "b2bware": {
      "url": "https://mcp.b2bware.com/sse",
      "headers": {
        "Authorization": "Bearer YOUR_B2BWARE_API_KEY"
      }
    }
  }
}
```

<Steps>
  <Step title="Open or create the config file" icon="file">
    Create `mcp.json` in your Cursor config directory if it doesn't exist.
  </Step>
  <Step title="Paste the configuration" icon="clipboard">
    Replace `YOUR_B2BWARE_API_KEY` with your actual API key.
  </Step>
  <Step title="Restart Cursor" icon="refresh-cw">
    Restart Cursor to load the MCP server. You should see the B2Bware tools in the MCP panel.
  </Step>
</Steps>

  </Tab>

  </Step>

  </Tab>


Supported languages: `de` (German), `en` (English), `fr` (French), `es` (Spanish)

## Security

<Callout kind="alert" title="Connection Isolation">
Every connection is fully isolated by your API key. The server stores no keys — they only live in memory for the duration of the active session and are discarded immediately on disconnect.
</Callout>

- API keys are never persisted on the MCP server
- Each session is isolated — no cross-session data leakage
- Prefer header-based authentication over URL parameters
- Never commit API keys to version control — use environment variables or secure input prompts

## Troubleshooting

<Expandable title="Server not connecting" default-open="false">

1. Verify your API key is correct (test it with a direct API call first)
2. Check the server status at [mcp.b2bware.com](https://mcp.b2bware.com)
3. Ensure your client supports SSE (Server-Sent Events) transport
4. Restart your AI tool after changing the configuration

</Expandable>

<Expandable title="Tools not appearing" default-open="false">

1. Check that the config file is in the correct location for your tool
2. Verify the JSON syntax is valid (no trailing commas, correct quotes)
3. For Windsurf: use `serverUrl` instead of `url`
4. Restart the application after saving the config

</Expandable>
