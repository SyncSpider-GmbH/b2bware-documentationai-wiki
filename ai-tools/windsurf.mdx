## Windsurf Integration

[Windsurf](https://codeium.com/windsurf) (by Codeium) reads `.windsurfrules` from your project root. It also supports MCP servers for live data access.

## Setup

<Steps>
  <Step title="Download .windsurfrules" icon="download">

```bash
curl -o .windsurfrules https://raw.githubusercontent.com/SyncSpider-GmbH/b2bware-documentationai-wiki/main/.cursorrules
```

<Callout kind="info">
The `.cursorrules` content is compatible with Windsurf's rule format.
</Callout>

  </Step>
  <Step title="Add MCP Server (optional)" icon="server">

Create or edit the Windsurf MCP config:

**Mac:** `~/.codeium/windsurf/mcp_config.json`
**Windows:** `%USERPROFILE%\.codeium\windsurf\mcp_config.json`

```json
{
  "mcpServers": {
    "b2bware": {
      "serverUrl": "https://mcp.b2bware.com/sse",
      "headers": {
        "Authorization": "Bearer YOUR_B2BWARE_API_KEY"
      }
    }
  }
}
```

<Callout kind="alert" title="Important">
Windsurf uses `serverUrl` (not `url`). This is different from Cursor and Claude Desktop.
</Callout>

  </Step>
  <Step title="Restart Windsurf" icon="refresh-cw">

Restart Windsurf to load the rules and MCP server.

  </Step>
</Steps>

## Scoped Rules (Advanced)

Windsurf supports workspace rules in `.windsurf/rules/` (max 12,000 characters per file):

```
.windsurf/
└── rules/
    ├── datahub-api.md         # General API conventions
    └── datahub-products.md    # Product-specific patterns
```

## Example Prompts

| Prompt | What Windsurf does |
|--------|-------------------|
| "Write a DataHub API client in Python" | Generates a client class with auth, pagination, and error handling |
| "Search for products with low stock" | Uses `filter[without_stock]` and stock quantity filters |
| "Create a bulk price update script" | Uses `POST /prices/upsert/bulk` with correct validation |
