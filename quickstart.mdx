## Prerequisites

Before you begin, you need:
- A B2Bware DataHub installation (your DataHub administrator provides this).
- An **installation key** — a UUID identifying the installation. Safe to use in browser code.
- An optional **API token** if you are building a backend integration rather than a storefront. See [Authentication](/authentication) for the full overview of credentials.

## Your First API Call

The example below uses the **storefront mode** (`installation-key`), which is what you want for browser- or mobile-based shops. For server-to-server integrations, replace the storefront header with `Authorization: Bearer YOUR_API_TOKEN` — see [Server-to-Server Tokens](/authentication/server-to-server).

<Steps>
  <Step title="Set up your installation key" icon="key">
    Store the installation key for use in requests:

```bash
export DATAHUB_INSTALLATION_KEY="7f1c9a40-2b3d-4e5f-9a8b-1c2d3e4f5061"
export DATAHUB_DOMAIN="api.datahub.syncspider.com"
```
  </Step>
  <Step title="List products" icon="box">
    Fetch the first page of public products with pricing and images:

    <Request tabs="cURL">
```bash
curl -s -X GET \
  "https://${DATAHUB_DOMAIN}/api/v1/apps/product-hub/products?filter[status]=public&include=price,finalPrice,mainMedia&per_page=5&sort=-_created_at" \
  -H "installation-key: ${DATAHUB_INSTALLATION_KEY}" \
  -H "Accept: application/json"
```
    </Request>
  </Step>
  <Step title="Log in a customer" icon="user">
    Exchange email and password for an `x-auth-token`:

    <Request tabs="cURL">
```bash
curl -s -X POST \
  "https://${DATAHUB_DOMAIN}/api/v1/apps/auth-hub/login" \
  -H "installation-key: ${DATAHUB_INSTALLATION_KEY}" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{ "email": "jane.doe@example.com", "password": "your-password" }'
```
    </Request>

    Save the `token` value from the response — it is the `x-auth-token` for the next step.

```bash
export DATAHUB_AUTH_TOKEN="42.JDJ5JDEyJGFiY2RlZmdo..."
```
  </Step>
  <Step title="Fetch the logged-in customer's cart" icon="shopping-cart">
    With both headers present, the cart endpoints return data scoped to the customer:

    <Request tabs="cURL">
```bash
curl -s -X GET \
  "https://${DATAHUB_DOMAIN}/api/v1/apps/order-hub/carts?include=cartItems.product.price,cartItems.product.mainMedia" \
  -H "installation-key: ${DATAHUB_INSTALLATION_KEY}" \
  -H "x-auth-token: ${DATAHUB_AUTH_TOKEN}" \
  -H "Accept: application/json"
```
    </Request>
  </Step>
</Steps>

## Understanding the Response

All list endpoints return paginated responses:

```json
{
  "data": [
    {
      "id": 1,
      "name": "Premium Widget",
      "sku": "WDG-001",
      "status": "public",
      "price": { "price": 29.99, "price_type": "regular" },
      "finalPrice": { "price": 24.99 },
      "mainMedia": { "media_url": "https://cdn.example.com/widget.jpg" }
    }
  ],
  "current_page": 1,
  "last_page": 10,
  "per_page": 5,
  "total": 50
}
```

## Common Operations

| Operation | Hub | Endpoint |
|-----------|-----|----------|
| List products | ProductHub | `GET /api/v1/apps/product-hub/products` |
| Search customers (admin) | CustomerHub | `GET /api/v1/apps/customer-hub/customers?filter[search]=term` |
| Check stock | ProductHub | `GET /api/v1/apps/product-hub/stock?filter[product_id]=...` |
| Calculate tax | TaxHub | `POST /api/v1/apps/tax-hub/calculate` |
| Log in a customer | AuthHub | `POST /api/v1/apps/auth-hub/login` |
| Create an order | OrderHub | `POST /api/v1/apps/order-hub/orders` |
