## Overview

ProductHub is the most complex hub in the DataHub platform. It manages products with complex pricing (regular, special, tier, group, cost), multi-warehouse stock, configurable/variation products, category trees with anchor categories, media management, cross-sells/up-sells, vendor relationships, AI-generated descriptions, translations, and deep attribute filtering.

**Base URL:** `api/v1/apps/product-hub/`

<Callout kind="info">
All endpoints require Bearer token authentication. See [Authentication](/authentication).
For filter syntax, pagination, and sorting, see [Common Patterns](/core-concepts/common-patterns).
</Callout>

**Standard Filters**

| # | Filter Key | Type | Description |
|---|------------|------|-------------|

|---|------------|------|-------------|

| 10 | `stock.stock_quantity` | number | Filter by stock quantity via stock relation |

| 12 | `price` | number | Filter by regular price (supports range) |

| 16 | `attributeValues.value.attribute_value` | text | Filter by attribute value text |
| 17 | `attributeValues.attribute_value_id` | number | Filter by specific attribute value ID |
| 18 | `categories.category_id` | number | Filter by exact category (no descendants) |

**Special Callback Filters**

| # | Filter Key | Type | Description |
|---|------------|------|-------------|
| 19 | `attribute` | JSON array | Advanced attribute filtering via `AttributeFilterRepository` (see below) |
| 20 | `exclude_child_products` | callback | Set to `1` to exclude child variations |
| 21 | `exclude_parent_products` | callback | Set to `1` to exclude configurable parents |
| 22 | `without_stock` | callback | Set to `1` to exclude products with no stock records |
| 23 | `exclude_child_products_id` | callback | Parent product ID to exclude its children |
| 24 | `parent_id` | callback | Parent product ID to return only its children |
| 25 | `search` | callback | Full-text search across name, sku, description, short_description |
| 26 | `anchor_category_id` | callback | Filter by category and all its descendant categories |

**Attribute Filter Format (Filter #19)**

The `attribute` filter accepts a JSON-encoded array:

```
filter[attribute]=[{"attribute_id":1,"attribute_value":"Red","condition":"IN","operator":"AND"}]
```

| Field | Values | Description |
|-------|--------|-------------|
| `attribute_id` | number | The attribute definition ID |
| `attribute_value` | string | Value(s) to match — comma-separated for multi-value |
| `condition` | `IN`, `NOT_IN`, `EQUALS`, `NOT_EQUALS`, `LIKE`, `GT`, `LT`, `GTE`, `LTE` | Match condition |
| `operator` | `AND`, `OR` | How to combine with other attribute conditions |
