## Overview

The AttributesHub manages dynamic product attributes, including attribute definitions, types, values, and groups. It supports polymorphic attributable relations and translations, allowing you to extend products and other entities with custom data.

**Base URL:** `api/v1/apps/attributes-hub/`

<Callout kind="info">
All endpoints require Bearer token authentication. See [Authentication](/authentication).
For filter syntax, pagination, and sorting, see [Common Patterns](/core-concepts/common-patterns).
</Callout>

## Endpoints

## Validation Rules

<Expandable title="Create Attribute — POST /attributes" default-open="true">

| Field | Rules |
|-------|-------|
| `name` | required, string, unique per type |
| `type` | required, string (one of available types) |
| `group_id` | required, numeric |
| `display_options` | sometimes, json |
| `visible_on_the_product_page` | sometimes, boolean |
| `used_for_filtering` | sometimes, boolean |
| `filterable_with_results` | sometimes, boolean |
| `required` | sometimes, boolean |
| `position` | sometimes, numeric |
| `meta_data` | sometimes, array |

</Expandable>

<Expandable title="Create Attribute Value — POST /attributeValues" default-open="false">

| Field | Rules |
|-------|-------|
| `attribute_id` | required, numeric |
| `value` | required |
| `meta_data.definition_values` | sometimes, array |

</Expandable>

<Expandable title="Create Attribute Group — POST /attributeGroups" default-open="false">

| Field | Rules |
|-------|-------|
| `name` | required, string |
| `type` | required, string, in: `dynamic` |
| `display_order` | sometimes, numeric |

</Expandable>

<Expandable title="Bulk Create/Update — PUT /attributes/insert/bulk" default-open="false">

Each item in the array requires: `name` (required, string), `type` (required, string), `group_id` (required, numeric), plus all optional fields from the single create endpoint.

</Expandable>

## Examples

<Steps>
  <Step title="List attributes with filter and include">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/attributes-hub/attributes?filter[type]=[{"condition":"=","value":"select"}]&include=values,group&per_page=10&sort=position' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Create a new attribute">
    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://api.datahub.syncspider.com/api/v1/apps/attributes-hub/attributes' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "name": "Material",
    "type": "select",
    "group_id": 1,
    "visible_on_the_product_page": true,
    "used_for_filtering": true,
    "filterable_with_results": true,
    "required": false,
    "position": 5
  }'
```
    </Request>
  </Step>
  <Step title="Get attribute values for a specific attribute">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/attributes-hub/attributeValues?filter[attribute_id]=[{"condition":"=","value":42}]&include=attribute,translation&sort=value' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Bulk create attributes">
    <Request tabs="cURL">
```bash
curl -s -X PUT \
  'https://api.datahub.syncspider.com/api/v1/apps/attributes-hub/attributes/insert/bulk' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '[
    {"name": "Weight", "type": "numeric", "group_id": 2, "position": 1},
    {"name": "Dimensions", "type": "text", "group_id": 2, "position": 2},
    {"name": "Color", "type": "swatch", "group_id": 1, "position": 3, "visible_on_the_product_page": true, "used_for_filtering": true}
  ]'
```
    </Request>
  </Step>
  <Step title="Filter visible and filterable attributes">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/attributes-hub/attributes?filter[visible_on_the_product_page]=[{"condition":"=","value":true}]&filter[used_for_filtering]=[{"condition":"=","value":true}]&include=values.translation,group&sort=-position' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
</Steps>
