## Overview

The NotificationHub is the most feature-rich hub with 120+ endpoints. It provides complete notification lifecycle management across multiple channels (email, SMS, push, Slack, broadcast, database), notification types with templates, user preferences, delivery attempt tracking, and a comprehensive analytics dashboard.

**Base URL:** `api/v1/apps/notification-hub/`

<Callout kind="info">
All endpoints require Bearer token authentication. See [Authentication](/authentication).
For filter syntax, pagination, and sorting, see [Common Patterns](/core-concepts/common-patterns).
</Callout>

## Supported Channels

| Channel | Description |
|---------|-------------|
| `email` | Email delivery |
| `database` | In-app database storage |
| `broadcast` | Real-time broadcasting (WebSocket) |
| `sms` | SMS text message |
| `push` | Push notification |
| `slack` | Slack message |

## Examples

<Steps>
  <Step title="List failed notifications with includes">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/notification-hub/notifications?filter[status]=[{"condition":"=","value":"failed"}]&include=notificationType,attempts,creator&per_page=20&sort=-_created_at' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Create a notification for a specific user">
    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://api.datahub.syncspider.com/api/v1/apps/notification-hub/notifications/create-for-user' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "notification_type_id": 5,
    "notifiable_type": "Customers",
    "notifiable_id": 1042,
    "title": "Order Shipped",
    "message": "Your order #ORD-2024-8847 has been shipped.",
    "channels": ["email", "database", "push"],
    "priority": 7,
    "data": {
      "order_id": 8847,
      "tracking_number": "1Z999AA10123456784",
      "carrier": "UPS"
    }
  }'
```
    </Request>
  </Step>
  <Step title="Get dashboard overview">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/notification-hub/dashboard/overview' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Filter delivery attempts by channel">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://api.datahub.syncspider.com/api/v1/apps/notification-hub/attempts?filter[channel]=[{"condition":"=","value":"email"}]&filter[status]=[{"condition":"IN","value":["failed","retry"]}]&include=notification,notification.notificationType&sort=-attempted_at&per_page=50' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Test a notification channel connection">
    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://api.datahub.syncspider.com/api/v1/apps/notification-hub/settings/email/test' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "recipient": "admin@example.com",
    "subject": "Test Notification",
    "message": "This is a test to verify the email channel configuration."
  }'
```
    </Request>
  </Step>
</Steps>
