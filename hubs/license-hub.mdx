## Overview

The LicenseHub provides complete license key lifecycle management including generation, activation, deactivation, revocation, pools, rules, usage tracking, and configurations. It enables you to create and distribute software license keys, organize them into pools, enforce activation rules, track usage, and configure license behavior per product.

**Base URL:** `api/v1/apps/license-hub/`

<Callout kind="info">
All endpoints require Bearer token authentication. See [Authentication](/authentication).
For filter syntax, pagination, and sorting, see [Common Patterns](/core-concepts/common-patterns).
</Callout>

</Expandable>

<Expandable title="Generate License Keys â€” POST /license-keys/generate" default-open="false">

| Field | Rules |
|-------|-------|
| `product_id` | required, integer |
| `template` | required, string |
| `allowed_chars` | sometimes, string |
| `quantity` | required, integer, min: 1, max: 1000 |
| `mode` | sometimes, in: `quantity_per_licence`, `single_licence_quantity` |

<Callout kind="tip">
Use `X` characters in the template as placeholders for random characters. For example, `XXXX-XXXX-XXXX-XXXX` generates keys like `A3F7-K9B2-M1D4-P6Q8`. The `allowed_chars` field lets you restrict which characters are used.
</Callout>

</Expandable>

## Examples

<Steps>
  <Step title="List active license keys with includes">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://your-domain.com/api/v1/apps/license-hub/license-keys?filter[status]=[{"condition":"=","value":"activated"}]&include=usageLogs,pool,product&sort=-expiration_date&per_page=25' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Generate license keys in bulk">
    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://your-domain.com/api/v1/apps/license-hub/license-keys/generate' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "product_id": 12,
    "template": "XXXX-XXXX-XXXX-XXXX",
    "quantity": 100,
    "mode": "quantity_per_licence"
  }'
```
    </Request>
  </Step>
  <Step title="Activate a license key">
    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://your-domain.com/api/v1/apps/license-hub/license-keys/157/activate' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "ip_address": "192.168.1.100",
    "domain": "client-app.example.com"
  }'
```
    </Request>
  </Step>
  <Step title="Get usage logs for a specific key">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://your-domain.com/api/v1/apps/license-hub/license-usage-logs/license-key/157?include=licenseKey&sort=-created_at&per_page=50' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
  <Step title="Filter keys expiring soon">
    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://your-domain.com/api/v1/apps/license-hub/license-keys?filter[status]=[{"condition":"IN","value":["activated","assigned"]}]&filter[expiration_date]=[{"condition":">=","value":"2026-03-19"},{"condition":"<=","value":"2026-04-19"}]&include=product,pool&sort=expiration_date' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
</Steps>
