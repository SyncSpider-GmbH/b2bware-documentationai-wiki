## Overview

This guide covers how to bulk import products into DataHub using the ProductHub API endpoints.

## Import Methods

| Method | Endpoint | Best For |
|--------|----------|----------|
| Single upsert | `PUT /products/upsert/single` | Ongoing sync, small batches |
| Bulk update | `PUT /products/bulk/update` | Updating existing products |
| Direct create | `POST /products` | One-off imports |
| Demo data | `POST /demoData/deploy` | Testing and development |

## Step-by-Step Import

<Steps>
  <Step title="Prepare your data" icon="file-text">
    Ensure each product has at minimum:
    - `name` (string, min 2 characters)
    - `sku` (string, unique across all products)
    - `product_type` (simple, configurable, license, or dt-one)
    - `status` (draft, private, or public)
  </Step>
  <Step title="Create categories first" icon="folder">
    If your products need categories, create them before importing products:

    <Request tabs="cURL">
```bash
curl -s -X POST \
  'https://your-domain.com/api/v1/apps/product-hub/categories' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{"name": "Electronics", "slug": "electronics"}'
```
    </Request>
  </Step>
  <Step title="Import products using upsert" icon="upload">
    Use the upsert endpoint for each product. This handles both creation and updates:

    <Request tabs="cURL">
```bash
curl -s -X PUT \
  'https://your-domain.com/api/v1/apps/product-hub/products/upsert/single' \
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{
    "action": "createOrUpdate",
    "sku": "PROD-001",
    "name": "My Product",
    "product_type": "simple",
    "status": "public",
    "categories": [1, 5],
    "prices": {"price": 29.99},
    "stock": [{"warehouse_id": 1, "stock_quantity": 50}]
  }'
```
    </Request>
  </Step>
  <Step title="Verify the import" icon="check-circle">
    List your imported products to verify:

    <Request tabs="cURL">
```bash
curl -s -X GET \
  'https://your-domain.com/api/v1/apps/product-hub/products?include=price,stock,categories.category&sort=-_created_at&per_page=10' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```
    </Request>
  </Step>
</Steps>

## Tips

- Process products in batches of 50-100 for optimal performance
- Use `auto_create_value: true` in attributes to automatically create attribute values
- Start with `status: "draft"` and switch to `public` after verification
- Use the `external_id` or `sku` field to track products from your source system
